Базовые элементы работы с git.

Git - это система котнроля версий, позволяющая "контролировать версии".
Мы используем Git для совместной работы, для отследивания изменений кода, для возможности возвращаться к любому ранее сохраненному состоянию.

Давайте с вами создадим проект, который будем использовать на нескольких ближайших занятиях.

Для хранения проекта мы будем использовать github.com

Если у вас нет акаунта, то перейдите на github.com и зарегистрируйтесь.

Мы готовы создать первый проект.

На github.com
Создаем хранилище: большая белая кнопка Start project.
Вписываем название. Нажимаем еще одну зеленую кнопку "Create repository"

У вас появился репозиторий - хранилище для вашего кода. 

Подключаем его к RStudio.

Сверху вы видите ссылку на ваш репозиторий и один из двух режимов доступа https и ssh.
Нас интересует https. Скопируйте соответствующую ссылку.

Теперь создадим проект в RStudio связанный с этим репозиторием.

Выбираем из меню File -> New project... -> Version control -> Git

 
Вставляем ссылку которую мы копировали с github в поле Repository URL:
Меняем если надо название каталога в котором будет храниться проект.
Меняем если надо локальный каталог в котором мы будем хранить проекты.
Нажимаем Create project.

У нас появился проект. 
Давайте скопируем в эту папку файл ~/shared/minor3_2017/4-Project/Practice22-Rec-Shiny-Git/Recommendation+Shiny.Rmd

Мы провели какие-то манипуляции с кодом и считаем, что текущее состояние осмысленно. Давайте зафиксируем его.

Для этого нажимаем кнопку git в верхней части экрана и в выпадающем меню нажимаем commit.
Выбираем файлы изменение которых хотим зафиксировать, пишем комментарий и нажимаем кнопку Commit

Система может пожаловаться, что у вас не настроены имя и email. Чтобы это исправить открываем Tools -> Shell

И вводим:
git config --global user.name "Тут надо вписать ваше имя"
git config --global user.email "Тут надо вписать вашу почту"


Либо вы можете вызвать из R команду  system("git config --global user.name \"Тут ваше имя\"")
